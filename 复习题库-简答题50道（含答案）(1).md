## 一、大数据概述（第 1—8 题）

### 1. 简述大数据"4V"特征的具体含义。
**参考答案：**
- 容量（Volume）：数据量庞大，通常以 TB/PB 为单位；
- 速度（Velocity）：数据产生和处理速度快，需要实时或近实时处理；
- 多样性（Variety）：数据类型多样，包括结构化、半结构化和非结构化数据；
- 价值（Value）：数据价值密度低，需要通过分析挖掘提取有价值信息。

### 2. 简述数据挖掘与数据分析的区别与联系，说明两者的核心目标。
**参考答案：**
- 区别：数据分析范围更广，涵盖数据收集、清洗、分析、可视化全过程，常用统计与可视化方法；数据挖掘是数据分析的子集，专注于从海量数据中发现隐藏模式，多使用机器学习与算法。
- 联系：数据挖掘是数据分析的高级阶段，数据分析为数据挖掘提供基础，两者互补。
- 核心目标：均是从数据中提取有价值的信息以支持决策。

### 3. 简述结构化、半结构化、非结构化数据的区别，并各举一例。
**参考答案：**
- 结构化数据：具备固定字段与表结构，如关系数据库表；
- 半结构化数据：具备部分结构性标签但无统一模式，如 XML、JSON、HTML；
- 非结构化数据：无固定结构，如图片、音频、视频、纯文本。

### 4. 简述大数据处理的基本流程。
**参考答案：**
（1）数据采集；（2）数据存储；（3）数据预处理；（4）数据分析与挖掘；（5）数据可视化；（6）数据应用与决策。

### 5. 简述大数据的主要应用场景（至少列举 4 个）。
**参考答案：**
- 金融风控与精准营销；
- 智慧医疗与公共卫生；
- 智能交通与城市管理；
- 推荐系统与电商个性化；
- 工业物联网与预测性维护。

### 6. 简述大数据时代面临的主要挑战。
**参考答案：**
- 海量数据的存储与计算成本；
- 数据质量与价值密度低；
- 数据安全与隐私保护；
- 数据多样性带来的异构整合难题；
- 实时性处理与人才短缺问题。

### 7. 简述数据价值密度低的含义及应对方法。
**参考答案：**
- 含义：单位数据中蕴含的有用信息相对较少，大部分数据对决策没有直接价值。
- 应对：通过数据清洗去噪、数据规约压缩、特征工程与数据挖掘等手段提取关键信息。

### 8. 简述我国国家数据局成立的背景与职责。
**参考答案：**
国家数据局于 2023 年成立，主要负责数据资源的统筹规划、共享开放、产权制度建设、数字经济与数字社会发展规划等，旨在推动数据要素市场化配置与高质量发展。

## 二、爬虫与大数据（第 9—14 题）

### 9. 简述网络爬虫的基本原理及使用 Requests 和 BeautifulSoup 库的核心步骤。
**参考答案：**
- 基本原理：模拟浏览器向目标网站发送 HTTP 请求，获取 HTML 源码，从中提取所需数据，实现自动化采集。
- 核心步骤：（1）安装 requests 与 BeautifulSoup；（2）导入库；（3）使用 requests.get() 发送请求并设置 User-Agent；（4）用 BeautifulSoup 解析 HTML；（5）通过选择器定位并提取目标数据；（6）整理并保存数据。

### 10. 简述常见的反爬手段及应对策略。
**参考答案：**
- 常见反爬：User-Agent 检测、IP 限频、验证码、登录态校验、动态渲染、字体反爬等；
- 应对策略：设置 User-Agent 与 Headers、使用代理 IP 池、控制请求频率、使用 Session 维持登录、引入 Selenium 处理动态页面、合理识别验证码或绕过检测。

### 11. 简述爬虫的合法性与 robots 协议的作用。
**参考答案：**
- 爬虫应遵守相关法律法规和网站的使用条款，不得爬取涉及隐私、商业秘密或著作权的数据；
- robots.txt 是网站用于声明哪些路径允许或禁止爬虫访问的协议文件，爬虫应主动遵守。

### 12. 简述 GET 与 POST 请求的区别。
**参考答案：**
- GET：参数附加在 URL 中，适合查询场景，可被缓存、有长度限制；
- POST：参数在请求体中，适合提交表单或上传数据，无明显长度限制，安全性相对较高；
- 在 requests 中分别对应 requests.get() 与 requests.post()。

### 13. 简述使用 requests.Session 的好处。
**参考答案：**
- 自动管理 Cookie，可在多次请求间保持登录状态；
- 复用底层 TCP 连接，提高性能；
- 可统一设置请求头、代理等配置，代码更简洁。

### 14. 简述如何爬取动态渲染网页。
**参考答案：**
- 方式一：分析浏览器抓包，找到真正的 AJAX 数据接口，直接用 requests 请求；
- 方式二：使用 Selenium、Playwright 等浏览器自动化工具，等待页面渲染完成后再提取 DOM 数据。

## 三、Scrapy 爬虫（第 15—18 题）

### 15. 简述 Scrapy 框架的核心组件及作用。
**参考答案：**
- Engine：核心引擎，控制数据流；
- Scheduler：调度器，管理请求队列；
- Downloader：下载器，负责发送请求并获取响应；
- Spider：爬虫，定义抓取逻辑和解析方法；
- Item Pipeline：管道，处理与持久化数据；
- Middlewares：中间件，扩展请求/响应处理流程。

### 16. 简述 Scrapy 项目的开发流程。
**参考答案：**
（1）scrapy startproject 创建项目；（2）在 items.py 中定义数据结构；（3）scrapy genspider 创建爬虫并编写解析逻辑；（4）在 pipelines.py 中实现数据持久化；（5）在 settings.py 中配置参数；（6）scrapy crawl 启动爬虫。

### 17. 简述 Scrapy 中 Item 和 Pipeline 的关系。
**参考答案：**
- Item 是数据结构容器，定义需要抓取的字段；
- Spider 解析后生成 Item，将其交给 Pipeline；
- Pipeline 负责对 Item 进行清洗、去重、验证以及保存到文件或数据库。

### 18. 简述 Scrapy 相比 requests + BeautifulSoup 的优势。
**参考答案：**
- 异步并发，效率高；
- 内置调度、去重与中间件机制；
- 项目结构规范，便于维护与扩展；
- 支持丰富的扩展点（下载延迟、代理、Cookie 处理、断点续爬等）。

## 四、数据库连接与查询（第 19—24 题）

### 19. 简述MySQL数据库的基本特点及常用的SQL操作(至少列举5种)。
**参考答案：**
- 基本特点：（1）开源免费；（2）跨平台兼容性强；（3）性能高、处理速度快；（4）易用，SQL 语法简单；（5）安全性好，支持权限管理。
- 常用 SQL 操作：SELECT（查询）、INSERT（插入）、UPDATE（更新）、DELETE（删除）、CREATE TABLE（建表）、DROP TABLE（删表）。

### 20. 简述使用 pymysql 连接并查询 MySQL 数据库的基本步骤。
**参考答案：**
（1）import pymysql；（2）pymysql.connect() 建立连接，传入 host、user、password、db 等参数；（3）conn.cursor() 获取游标；（4）cursor.execute(sql) 执行 SQL；（5）cursor.fetchall() 或 fetchone() 获取结果；（6）conn.commit() 提交事务（写操作）；（7）cursor.close() 与 conn.close() 关闭资源。

### 21. 简述 SQL 中 WHERE 与 HAVING 的区别。
**参考答案：**
- WHERE：用于分组前对原始数据行进行过滤，不能使用聚合函数；
- HAVING：用于分组后对聚合结果进行过滤，可使用聚合函数（如 COUNT、SUM）；
- 二者可同时出现：先 WHERE 过滤行 → 再 GROUP BY 分组 → 再 HAVING 过滤分组。

### 22. 简述聚合函数 COUNT、SUM、AVG、MAX、MIN 的作用。
**参考答案：**
- COUNT：统计行数；
- SUM：求列总和；
- AVG：求列平均值；
- MAX：求列最大值；
- MIN：求列最小值。

### 23. 简述 SQL 注入及防范措施。
**参考答案：**
- SQL 注入：攻击者通过在输入中拼接恶意 SQL，绕过校验或破坏数据；
- 防范：使用参数化查询/预编译语句、严格校验输入、最小权限原则、使用 ORM 框架、避免拼接 SQL。

### 24. 简述使用 Pandas 与 MySQL 交互的常用方法。
**参考答案：**
- 读取：pd.read_sql(sql, conn) 将查询结果加载为 DataFrame；
- 写入：df.to_sql(table_name, conn, if_exists='append') 将 DataFrame 写入数据库；
- 优势：可直接利用 Pandas 进行分析，无需手动遍历游标结果。

## 六、大数据存储与清洗（第 32—37 题）

### 32. 简述数据预处理的主要步骤及各步骤的核心作用。
**参考答案：**
- （1）数据采集：获取符合分析需求的原始数据；
- （2）数据清洗：处理错误、缺失、重复、异常问题，提高数据质量；
- （3）数据转换：标准化、离散化、归一化等，统一数据格式；
- （4）数据集成：将多数据源合并为统一数据集；
- （5）数据规约：在保留关键信息前提下减少数据量，提高效率。

### 33. 数据清洗的主要任务是什么?请列举数据清洗的常见方法。
**参考答案：**
- 主要任务：去除重复数据、处理缺失值、纠正错误数据、统一数据格式。
- 常见方法：数据过滤（按条件保留有效数据）、数据转换（格式转换）、数据填充（均值/中位数/众数填充缺失值）、数据验证（约束规则验证）。

### 34. 简述常见的缺失值处理方法及适用场景。
**参考答案：**
- 删除法：缺失比例小且对样本量影响不大时使用；
- 均值/中位数填充：数值型数据，分布较均匀；
- 众数填充：类别型数据；
- 插值法：时间序列、连续变化数据；
- 模型预测填充：缺失对结果影响较大时使用机器学习方法预测填补。

### 35. 简述数据标准化、归一化与离散化的区别。
**参考答案：**
- 标准化：通过减均值除标准差使数据具有零均值与单位方差（Z-Score）；
- 归一化：按比例缩放，使数据落入 [0,1] 等特定区间（Min-Max）；
- 离散化：将连续值划分到若干区间，转换为离散类别（如分箱）。

### 36. 简述 ETL 的基本流程及与 ELT 的区别。
**参考答案：**
- ETL：抽取（Extract）→ 转换（Transform）→ 加载（Load），先转换再入库，适合传统数据仓库；
- ELT：抽取（Extract）→ 加载（Load）→ 转换（Transform），先入库再转换，利用大数据平台算力，适合数据湖或云数仓。

### 37. 简述关系型数据库与 NoSQL 数据库的区别。
**参考答案：**
- 关系型：行列结构、强 Schema、支持事务与 SQL，如 MySQL、Oracle；
- NoSQL：模式灵活、可水平扩展、高并发，包括键值（Redis）、文档（MongoDB）、列族（HBase）、图数据库（Neo4j）；
- 选型：结构稳定且强一致用关系型；海量、高并发或半结构化用 NoSQL。

## 七、数据格式与编码技术（第 38—41 题）

### 38. 简述结构化、半结构化、非结构化数据的常见存储格式。
**参考答案：**
- 结构化：CSV、关系数据库表、Parquet/ORC（列式）；
- 半结构化：JSON、XML、YAML；
- 非结构化：TXT、Word、图片、音视频文件等。

### 39. 简述 CSV、JSON、XML 三种文件格式的特点。
**参考答案：**
- CSV：以逗号分隔的纯文本表格，体积小、解析快、但无层级结构；
- JSON：键值对结构，支持嵌套，易读易解析，是 Web API 常用格式；
- XML：使用标签描述数据，可携带属性与层级结构，冗余度较高。

### 40. 简述 UTF-8、GBK、ASCII 编码的区别。
**参考答案：**
- ASCII：单字节，仅表示英文字符与基本控制符；
- GBK：双字节，主要表示简体中文，兼容 ASCII；
- UTF-8：变长 1~4 字节，全球通用，可表示中文等多语言，是互联网首选编码。

### 41. 简述行式存储与列式存储的区别。
**参考答案：**
- 行式存储（如 CSV、关系数据库）：按行组织，适合事务和单条增删改；
- 列式存储（如 Parquet、ORC）：按列组织，便于压缩和按列聚合分析，更适合大数据分析场景。

## 八、数据抽取与采集（第 42—45 题）

### 42. 简述数据采集的主要来源及常见工具。
**参考答案：**
- 来源：业务数据库、日志文件、网络爬虫、API 接口、传感器与物联网设备、第三方数据。
- 工具：Sqoop（数据库迁移）、Flume/Logstash（日志采集）、Kafka（消息队列）、Scrapy/Requests（网络爬虫）、MQTT（物联网）。

### 43. 简述全量抽取与增量抽取的区别。
**参考答案：**
- 全量抽取：每次抽取全部数据，简单但资源消耗大，适合数据量小或初次加载；
- 增量抽取：仅抽取新增或变化的数据，效率高，适合大数据场景；
- 增量依据通常是时间戳字段、主键变化、数据库日志（CDC）等。

### 44. 简述批处理与流处理的区别。
**参考答案：**
- 批处理：按周期处理已积累的数据，吞吐大但延迟较高，典型框架如 MapReduce、Spark；
- 流处理：对实时到达的数据逐条/小批处理，延迟低，典型框架如 Flink、Spark Streaming、Kafka Streams。

### 45. 简述衡量数据采集质量的指标。
**参考答案：**
- 完整性：是否覆盖所需字段与样本；
- 准确性：是否真实反映源数据；
- 一致性：跨来源数据是否口径统一；
- 及时性：数据从产生到可用的延迟；
- 唯一性：是否存在重复数据。

## 九、Pandas 数据分析与清洗（第 46—50 题）

### 46. 简述 Pandas 中 Series 和 DataFrame 的区别与联系。
**参考答案：**
- 区别：（1）维度不同：Series 一维，DataFrame 二维；（2）索引不同：Series 仅有行索引，DataFrame 有行和列索引；（3）用途不同：Series 存储单维数据，DataFrame 存储多维结构化数据。
- 联系：（1）DataFrame 由多个 Series 组成，每列即一个 Series；（2）两者均可进行索引、切片、过滤等操作；（3）Series 可组合成 DataFrame，DataFrame 也可提取列作为 Series。

### 47. 试列举3种DataFrame的常用数据修改操作(如添加列、修改值等)并简述操作方法。
**参考答案：**
- 添加列：直接赋值 df['城市']=['北京','上海','广州']，或基于已有列计算 df['出生年份']=2024-df['年龄']，或使用 df.insert() 指定位置插入；

### 50. 简述 Pandas 处理时间序列数据的常用方法。
**参考答案：**

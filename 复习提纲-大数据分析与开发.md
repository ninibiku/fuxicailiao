## 一、大数据概述

### 1.1 大数据"4V"特征 🔑✏🔘

| 特征 | 英文 | 含义 |
|------|------|------|
| 容量 | Volume | 数据量庞大，以 TB/PB 为单位 |
| 速度 | Velocity | 数据产生和处理速度快，需实时/近实时处理 |
| 多样性 | Variety | 含结构化、半结构化、非结构化数据 |
| 价值 | Value | 价值密度低，需挖掘才能提取有用信息 |

### 1.2 数据结构分类 🔑✏🔘

| 类型 | 特点 | 举例 |
|------|------|------|
| 结构化 | 固定字段与表结构 | 关系数据库表、CSV |
| 半结构化 | 有标签但无统一模式 | JSON、XML、HTML |
| 非结构化 | 无固定结构 | 图片、音频、视频、纯文本 |

### 1.3 大数据处理流程 🔑✏🔘

```
数据采集 → 数据存储 → 数据预处理 → 数据分析与挖掘 → 数据可视化 → 数据应用与决策
```

### 1.4 数据分析 vs 数据挖掘 🔑🔘

- **数据分析**：范围更广，涵盖收集、清洗、分析、可视化全过程
- **数据挖掘**：数据分析的高级阶段 / 子集，专注于发现隐藏模式与潜在规律
- **共同目标**：从数据中提取有价值信息以支持决策

### 1.5 其他必记要点 ✏🔘

- 大数据最显著特征：**数据量巨大**
- 我国**国家数据局**成立于 **2023** 年，负责数据资源统筹规划
- "数据是新石油"强调数据的 **价值**
- 大数据生命周期起点：**数据采集**；终点：**数据应用/数据可视化**
- 主要挑战：存储成本、数据质量、安全隐私、异构整合、实时处理、人才短缺
- 应用场景：金融风控、智慧医疗、智能交通、推荐系统、工业物联网

## 二、爬虫与大数据

### 2.1 Requests + BeautifulSoup 核心流程 🔑✏🔘

```
导入库 → requests.get()发送请求(设置User-Agent) → BeautifulSoup解析HTML
       → 选择器定位提取数据 → 整理保存数据
```

### 2.2 核心方法与属性 ✏🔘

| 操作 | 代码/方法 |
|------|-----------|
| GET 请求 | `requests.get(url, params=..., headers=...)` |
| POST 请求 | `requests.post(url, data=...)` |
| 获取响应文本 | `response.text` |
| 获取字节流 | `response.content` |
| 获取 JSON | `response.json()` |
| 抛出 HTTP 异常 | `response.raise_for_status()` |
| 设置编码 | `response.encoding = 'utf-8'` |
| 设置超时 | `timeout` 参数 |
| 会话保持 | `requests.Session()` |

### 2.3 BeautifulSoup 核心方法 ✏🔘

| 操作 | 方法 |
|------|------|
| 查找所有匹配标签 | `soup.find_all('a')` |
| CSS 选择器查找 | `soup.select('...')` |
| 获取标签内文本 | `tag.text` 或 `tag.get_text()` |
| 获取属性值 | `tag.get('href')` |
| 常用解析器 | `html.parser` |

### 2.4 爬虫关键概念 🔑✏🔘

- **本质**：模拟浏览器向服务器发送请求并解析响应
- **核心步骤**：发送请求 → 获取响应 → 解析数据 → 存储数据
- **robots.txt**：网站声明爬虫允许/禁止访问路径的协议文件，爬虫应遵守
- **常见反爬**：User-Agent 检测、IP 限频、验证码、登录态校验、动态渲染、字体反爬
- **应对策略**：设置 Headers、代理 IP 池、控制请求频率、Session 维持登录、Selenium 处理动态页面

### 2.5 GET vs POST 🔑🔘

| | GET | POST |
|------|-----|------|
| 参数位置 | URL 中（`params`） | 请求体中（`data`） |
| 适用场景 | 查询 | 提交表单/上传 |
| 缓存 | 可缓存 | 不可缓存 |
| 长度限制 | 有 | 无明显限制 |

### 2.6 动态渲染网页 🔑🔘

- **方式一**：抓包找到 AJAX 数据接口，直接用 requests 请求
- **方式二**：使用 **Selenium** / Playwright 等浏览器自动化工具，等页面渲染完后提取 DOM

### 2.7 爬虫合法性 🔑🔘

- 遵守法律法规和网站使用条款
- 不得爬取隐私、商业秘密或著作权数据
- 遵守 robots.txt 协议

## 三、Scrapy 爬虫框架

### 3.1 五大核心组件 🔑✏🔘

| 组件 | 作用 |
|------|------|
| **Engine**（引擎） | 核心引擎，控制数据流 |
| **Scheduler**（调度器） | 管理请求队列 |
| **Downloader**（下载器） | 发送请求并获取响应 |
| **Spider**（爬虫） | 定义抓取逻辑和解析方法 |
| **Item Pipeline**（管道） | 处理与持久化数据 |
| **Middlewares**（中间件） | 扩展请求/响应处理流程 |

### 3.2 项目开发流程 🔑

```
① scrapy startproject 创建项目
② items.py 中定义数据结构
③ scrapy genspider 创建爬虫，编写解析逻辑
④ pipelines.py 中实现数据持久化
⑤ settings.py 中配置参数
⑥ scrapy crawl 爬虫名 启动爬虫
```

### 3.3 核心命令与文件 ✏🔘

| 操作 | 命令/文件名 |
|------|-------------|
| 创建项目 | `scrapy startproject` |
| 创建爬虫 | `scrapy genspider` |
| 启动爬虫 | `scrapy crawl` |
| 输出到文件 | `-o file.json` |
| 数据结构定义 | `items.py` |
| 数据持久化 | `pipelines.py` |
| 参数配置 | `settings.py` |
| 爬虫存放目录 | `spiders/` |

### 3.4 关键配置项 ✏🔘

| 配置项 | 含义 |
|--------|------|
| `ROBOTSTXT_OBEY` | 是否遵守 robots 协议 |
| `DOWNLOAD_DELAY` | 下载延迟 |
| `CONCURRENT_REQUESTS` | 并发请求数 |

### 3.5 Spider 要点 ✏🔘

- 爬虫类继承自 `scrapy.Spider`
- 起始 URL 列表名：`start_urls`
- 默认回调方法名：`parse`
- Item 是数据结构容器，Spider 生成 Item 交给 Pipeline 处理

### 3.6 Scrapy vs requests+BeautifulSoup 🔑

- 异步并发，效率高
- 内置调度、去重与中间件
- 项目结构规范，便于维护扩展

## 四、数据库连接与查询

### 4.1 MySQL 基本特点 🔑

- 开源免费、跨平台、高性能、SQL 语法简单、支持权限管理

### 4.2 pymysql 操作流程 🔑✏🔘

```
import pymysql → conn = pymysql.connect(host, user, password, db)
→ cursor = conn.cursor() → cursor.execute(sql)
→ cursor.fetchall() / fetchone() → conn.commit()（写操作）
→ cursor.close() → conn.close()
```

### 4.3 pymysql 核心对象与方法 ✏🔘

| 操作 | 方法/参数 |
|------|-----------|
| 连接主机 | `host` 参数 |
| 指定数据库 | `db` / `database` 参数 |
| 执行 SQL 的对象 | `cursor` |
| 获取所有记录 | `cursor.fetchall()` |
| 获取一条记录 | `cursor.fetchone()` |
| 提交事务 | `conn.commit()` |
| 回滚事务 | `conn.rollback()` |
| 关闭连接 | `conn.close()` |

| 分类 | 关键字 |

| 分组后过滤 | `HAVING` |
| 升序 | `ASC` |

### 4.5 聚合函数 ✏🔘

| 函数 | 作用 |
|------|------|
| `COUNT` | 统计行数 |
| `SUM` | 求和 |
| `AVG` | 求平均值 |
| `MAX` | 求最大值 |
| `MIN` | 求最小值 |

### 4.6 WHERE vs HAVING 🔑

- **WHERE**：分组前过滤原始行，不能使用聚合函数
- **HAVING**：分组后过滤聚合结果，可使用聚合函数
- 顺序：WHERE → GROUP BY → HAVING

### 4.7 数据库核心概念 🔑✏🔘

- **主键**：唯一且非空
- **外键**：建立表与表之间的关联
- **关系型数据库**：MySQL、Oracle、PostgreSQL（强 Schema、支持事务、SQL）
- **NoSQL**：Redis（键值）、MongoDB（文档）、HBase（列族）、Neo4j（图）

### 4.8 SQL 注入防范 🔑

- 使用**参数化查询/预编译语句**
- 严格校验输入、最小权限原则、使用 ORM 框架

## 六、大数据存储与清洗

### 6.1 数据预处理五大步骤 🔑✏🔘

| 步骤 | 核心作用 |
|------|----------|
| 数据采集 | 获取符合分析需求的原始数据 |
| 数据清洗 | 处理错误、缺失、重复、异常问题 |
| 数据转换 | 标准化、离散化、归一化，统一格式 |
| 数据集成 | 将多数据源合并为统一数据集 |
| 数据规约 | 保留关键信息前提下减少数据量 |

### 6.2 数据清洗 🔑✏🔘

- **主要任务**：去重、处理缺失值、纠正错误、统一格式
- **输出**：干净、可用的数据
- **处理重复数据**的核心目的：确保数据唯一性
- **数据噪声**：因不一致或错误导致的结果偏差
- **流程**：数据读取 → 数据检查 → 数据清洗 → 数据存储

### 6.3 缺失值处理方法 🔑✏🔘

| 方法 | 适用场景 |
|------|----------|
| 删除法 | 缺失比例小且对样本量影响不大 |
| 均值/中位数填充 | 数值型数据，分布均匀 |
| 众数填充 | 类别型数据 |
| 插值法 | 时间序列、连续变化数据 |
| 模型预测填充 | 缺失影响大时用 ML 预测 |

### 6.4 数据变换三种方式 🔑

| 方式 | 说明 |
|------|------|
| **标准化**（Z-Score） | 减均值除标准差，零均值单位方差 |
| **归一化**（Min-Max） | 缩放到 [0,1] 区间 |
| **离散化**（分箱） | 连续值→离散类别 |

### 6.5 ETL vs ELT 🔑✏🔘

| | ETL | ELT |
|------|-----|------|
| 顺序 | 抽取→转换→加载 | 抽取→加载→转换 |
| 特点 | 先转换再入库 | 先入库再利用平台算力转换 |
| 适用 | 传统数据仓库 | 数据湖、云数仓 |

### 6.6 大数据存储技术 ✏🔘

| 技术/概念 | 说明 |

## 七、数据格式与编码技术

### 7.1 三种数据格式对比 🔑✏🔘

| 格式 | 特点 |
|------|------|
| **CSV** | 逗号分隔纯文本表格，体积小、解析快，无层级结构（行式存储） |
| **JSON** | 键值对结构，支持嵌套，易读易解析，Web API 常用格式 |
| **XML** | 标签描述数据，可携带属性与层级结构，冗余度较高 |

### 7.2 编码方式 🔑✏🔘

| 编码 | 字节 | 范围 |
|------|------|------|
| **ASCII** | 单字节 | 仅英文字符与基本控制符，**不能表示中文** |
| **GBK** | 双字节 | 简体中文，兼容 ASCII |
| **UTF-8** | 变长 1~4 字节 | 全球通用，互联网首选编码 |

### 7.3 行式 vs 列式存储 🔑✏🔘

| | 行式存储 | 列式存储 |
|------|-----------|-----------|
| 代表格式 | CSV、关系数据库 | Parquet、ORC |
| 组织方式 | 按行 | 按列 |
| 适用场景 | 事务处理、单条增删改 | 大批量分析/聚合查询 |

### 7.4 Python 编码与文件操作 ✏🔘

| 操作 | 方法 |
|------|------|
| 字符串→字节 | `str.encode()` |
| 字节→字符串 | `bytes.decode()` |
| 读 CSV | `pd.read_csv('file.csv', encoding='utf-8')` |
| 读 Excel | `pd.read_excel('file.xlsx')` |
| 写 CSV | `df.to_csv('file.csv')` |
| 写 Excel | `df.to_excel('file.xlsx')` |

### 7.5 JSON 操作 ✏🔘

| 操作 | 方法 |
|------|------|
| 读取 JSON 文件 | `json.load(f)` |
| 写入 JSON 文件 | `json.dump(obj, f)` |
| JSON 字符串→字典 | `json.loads(s)` |
| 字典→JSON 字符串 | `json.dumps(obj)` |

### 7.6 其他要点 ✏🔘

- **Base64**：将二进制数据用文本表示
- 常见压缩算法：GZIP、Snappy、LZ4
- Avro 文件：用于序列化与跨平台传输

## 八、数据抽取与采集

### 8.1 数据采集来源与工具 🔑✏🔘

| 来源 | 工具/技术 |
|------|-----------|
| 业务数据库 | Sqoop（数据库↔Hadoop 迁移） |
| 日志文件 | Flume / Logstash |
| 消息队列 | Kafka（分布式消息中间件） |
| 网络爬虫 | Scrapy / Requests |
| API 接口 | HTTP 请求（返回 JSON） |
| 物联网设备 | MQTT 协议 |

### 8.2 全量抽取 vs 增量抽取 🔑✏🔘

| | 全量抽取 | 增量抽取 |
|------|-----------|----------|
| 方式 | 每次抽取全部数据 | 仅抽取新增/变化数据 |
| 优点 | 简单 | 效率高，资源消耗小 |
| 缺点 | 资源消耗大 | 实现复杂 |
| 适用 | 数据量小或初次加载 | 大数据场景 |
| 依据 | — | 时间戳、主键变化、日志（CDC） |

### 8.3 CDC 🔘

- **CDC** = Change Data Capture（变更数据捕获）

### 8.4 批处理 vs 流处理 🔑✏🔘

| | 批处理 | 流处理 |
|------|--------|--------|
| 方式 | 按周期处理已积累数据 | 实时逐条/小批处理 |
| 延迟 | 高 | 低 |
| 吞吐 | 大 | 较小 |
| 典型框架 | MapReduce、Spark | Flink、Spark Streaming、Kafka Streams |

### 8.5 数据采集质量指标 🔑✏🔘

| 指标 | 含义 |
|------|------|
| **完整性** | 是否覆盖所需字段与样本 |
| **准确性** | 是否真实反映源数据 |
| **一致性** | 跨来源数据口径是否统一 |
| **及时性** | 数据从产生到可用的延迟 |
| **唯一性** | 是否存在重复数据 |

### 8.6 其他要点 ✏🔘

- **ELK**：Elasticsearch + Logstash + Kibana，Logstash 负责日志采集与转发
- **幂等性**：多次执行结果一致
- 网页爬虫属于**主动式**数据采集
- 数据采集后的下一步通常是**数据清洗**
- 采集模式按时序分：**批处理**与**流式/实时处理**

## 九、Pandas 数据分析与清洗

### 9.1 Series vs DataFrame 🔑✏🔘

| | Series | DataFrame |
|------|--------|-----------|
| 维度 | 一维带索引数组 | 二维带行列索引表格 |
| 索引 | 仅行索引 | 行索引 + 列索引 |
| 关系 | — | 每列即一个 Series |

### 9.2 数据查看与统计方法（必记）✏🔘

| 方法 | 作用 |
|------|------|
| `df.head(n)` | 查看前 n 行 |
| `df.tail(n)` | 查看后 n 行 |
| `df.info()` | 行列结构、字段类型与缺失情况 |
| `df.describe()` | 均值、标准差、最值、分位数 |
| `df.shape` | 查看形状（行数, 列数） |
| `df.columns` | 列名 |
| `df.dtypes` | 列类型 |
| `df['列'].value_counts()` | 类别频数统计 |

### 9.3 数据处理核心方法 ✏🔘

| 操作 | 方法 |
|------|------|
| 删除重复数据 | `df.drop_duplicates()` |
| 填充缺失值 | `df.fillna(value)` |
| 删除缺失值 | `df.dropna()` |
| 判断缺失值 | `df.isna()` / `df.isnull()` |
| 按列分组 | `df.groupby('列名')` |
| 排序 | `df.sort_values('列名')` |
| 按索引排序 | `df.sort_index()` |
| 重置索引 | `df.reset_index()` |
| 设置索引列 | `df.set_index('列名')` |
| 修改列名 | `df.rename(columns={...})` |
| 替换数据 | `df.replace(old, new)` |
| 数据类型转换 | `df.astype(type)` |

### 9.4 数据选择 ✏🔘

| 方式 | 方法 |
|------|------|
| 按列名选择 | `df['列名']` |
| 按标签选择 | `df.loc[行标签, 列标签]` |
| 按位置选择 | `df.iloc[行位置, 列位置]` |
| 条件筛选 | `df[df['列'] > 值]` |

### 9.5 数据修改操作 🔑

| 操作 | 方法 |
|------|------|
| 添加列 | `df['新列'] = 值` 或 `df.insert(位置, '列名', 值)` |
| 修改值 | 直接赋值、`loc` 条件修改、`apply()` 函数 |
| 删除列 | `df.drop('列名', axis=1)` |
| 删除行 | `df.drop(行索引)` |

### 9.6 合并与聚合 ✏🔘

| 操作 | 方法 |
|------|------|
| 按键合并 | `pd.merge(df1, df2, on='键')` |
| 多键合并 | `on=['键1', '键2']` |
| 行/列拼接 | `pd.concat([df1, df2])` |
| 聚合多函数 | `df.groupby().agg({'列': ['sum', 'mean']})` |
| 透视表 | `df.pivot_table()` |
| 交叉表 | `pd.crosstab()` |
| 列均值 | `df.mean()` |
| 逐元素应用函数 | `df['列'].apply(func)` |
| Series 映射 | `df['列'].map(dict)` |

### 9.7 时间序列处理 🔑✏🔘

| 操作 | 方法 |
|------|------|
| 转为 datetime | `pd.to_datetime(df['日期'])` |
| 设为时间索引 | `df.set_index('日期')` |
| 提取年/月/日 | `df['日期'].dt.year / .month / .day` |
| 按月分组 | `df.groupby(df['日期'].dt.month)` |
| 按月重采样 | `df.resample('M').sum()` |
| 时间偏移 | `pd.DateOffset`、`shift()` |

### 9.8 相关库 🔘

| 库 | 用途 |
|------|------|
| **NumPy** | 数值计算基础库，高效数组操作 |
| **Pandas** | 数据分析与清洗核心库 |
| 点积计算 | `np.dot(a, b)` |

### 9.9 Python 数据分析基本流程 🔑

```
明确目标 → 数据采集 → 数据预处理（清洗/转换/集成/规约）
→ 探索性分析（统计描述+可视化）→ 分析建模 → 结果解释与报告
```

## 一、数据结构基础概念

### 1.1 数据结构三要素
- **逻辑结构**：数据元素之间的逻辑关系
- **存储结构**（物理结构）：数据结构在计算机中的存储表示
- **数据运算**：对数据元素施加的操作

> ⚠️ 逻辑结构独立于存储结构，同一逻辑结构可有多种存储实现。

### 1.2 逻辑结构的四种基本类型
| 类型 | 特点 | 示例 |
|------|------|------|
| 集合结构 | 元素同属一个集合，无其他关系 | 集合 |
| 线性结构 | 一对一的关系 | 线性表、栈、队列 |
| 树形结构 | 一对多的层次关系 | 树、二叉树 |
| 图形结构 | 多对多的关系 | 图 |

### 1.3 存储结构的两种基本类型
- **顺序存储**：连续内存空间，可随机存取，存储密度=1
- **链式存储**：非连续内存，通过指针连接，存储密度<1

### 1.4 数据层次关系
```
数据 > 数据元素 > 数据项
```
- **数据元素**：数据的基本单位
- **数据项**：数据的最小不可分割单位

### 1.5 抽象数据类型（ADT）
- 三元组表示：(数据对象, 数据关系, 基本操作集)
- 抽象操作的定义与具体实现无关

### 1.6 算法评价标准
- **时间复杂度**：与问题规模有关，与硬件/语言无关
- **空间复杂度**
- 算法的正确性、鲁棒性、可读性

## 二、线性表

### 2.1 线性表的定义
- n个数据元素的有限序列
- 除首元素（无前驱）和尾元素（无后继）外，每个元素有且仅有一个直接前驱和一个直接后继

### 2.2 顺序表 vs 链表

| 维度 | 顺序表 | 单链表 |
|------|--------|--------|
| 存储方式 | 连续内存 | 任意内存+指针 |
| 存取方式 | 随机存取 O(1) | 顺序存取 O(n) |
| 插入/删除 | 需移动元素 O(n) | 只需改指针 O(1)（已知位置）|
| 存储密度 | =1 | <1 |
| 适用场景 | 频繁查找、少插入删除 | 频繁插入删除 |

### 2.3 顺序表关键公式
- 第i个元素地址：`LOC(a₁) + (i-1) × L`（L为元素长度）
- 插入第i个位置需移动：`n - i + 1` 个元素
- 删除第i个位置需移动：`n - i` 个元素
- 插入/删除平均移动：`n/2` 个元素

### 2.4 单链表要点
- **头结点**：首元结点前的附加结点，方便运算（使空表和非空表处理统一）
- **首元结点**：存储第一个数据元素的结点
- **存储密度**：D/(D+N) < 1（D=数据域大小，N=指针域大小）
- 单链表中查找必须从头遍历，O(n)

### 2.5 头插法与尾插法
| 方法 | 插入位置 | 元素顺序 | 时间复杂度 |
|------|----------|----------|------------|
| 头插法 | 表头 | 与输入顺序相反（逆序）| O(1) |
| 尾插法 | 表尾 | 与输入顺序相同（正序）| O(1)有尾指针 / O(n)无尾指针 |

### 2.6 双链表与循环链表
- **双链表**：可访问前后相邻结点更灵活（牺牲空间换时间）
- **仅有尾指针的单循环链表**：在最后插入+删除第一个元素时最省时间

## 三、栈和队列

### 3.1 栈（Stack）— LIFO（后进先出）
- 插入/删除仅在**栈顶**进行
- **应用**：递归调用、函数调用、表达式求值、括号匹配、回文判断

### 3.2 队列（Queue）— FIFO（先进先出）
- 插入在**队尾**，删除在**队头**
- **应用**：缓冲区（如打印机缓冲区）、层次遍历

### 3.3 循环队列
- **队空条件**：`front == rear`
- **队满条件**：`(rear + 1) % m == front`（少用一个空间）
- **入队**：`rear = (rear + 1) % m`
- **出队**：`front = (front + 1) % m`
- 数组A[0..m]有m+1个元素，取模用m+1

### 3.4 栈和队列的共同点
- 都只允许在端点处插入和删除元素
- 都是操作受限的线性表

### 3.5 递归
- 递归算法必须包含：**终止条件** + **递归部分**

## 四、树与二叉树

### 4.1 树的基本定义
- 树是n(n≥0)个结点的有限集
- n=0为空树；非空树有且仅有一个根结点
- **结点的度**：结点拥有的子节点数目
- **树的度**：树中所有结点度的**最大值**（非平均值）
- **树的深度**：树中所有结点的最大层数

### 4.2 二叉树性质（★核心考点）

| 性质 | 内容 |
|------|------|
| 性质1 | 第k层最多有 $2^{k-1}$ 个结点（k≥1）|
| 性质2 | 深度为h的二叉树最多有 $2^h - 1$ 个结点 |
| 性质3 | **$n_0 = n_2 + 1$**（叶子数 = 度为2的结点数 + 1）|

> 🔑 **性质3是考试最高频考点！** 务必牢记。

### 4.3 满二叉树 vs 完全二叉树
- **满二叉树**：每层结点数都达到最大值
- **完全二叉树**：除最后一层外每层结点数达最大值，最后一层结点靠左排列
- **关系**：满二叉树 → 完全二叉树（反之不成立）

### 4.4 完全二叉树性质
- n为奇数时 $n_1 = 0$；n为偶数时 $n_1 = 1$
- 编号为i的结点，左孩子编号 $2i$，右孩子编号 $2i+1$
- 没有左孩子→必为叶子结点

### 4.5 二叉树遍历（★核心考点）

| 遍历方式 | 顺序 |
|----------|------|
| 先序遍历 | 根 → 左 → 右 |
| 中序遍历 | 左 → 根 → 右 |
| 后序遍历 | 左 → 右 → 根 |
| 层次遍历 | 逐层从左到右（借助**队列**）|

> 🔑 **已知两种遍历序列还原二叉树：必须包含中序序列！**
> - 先序+中序 → 可唯一确定二叉树 ✅
> - 后序+中序 → 可唯一确定二叉树 ✅
> - 先序+后序 → **不能**唯一确定二叉树 ❌

### 4.6 二叉树存储结构
| 存储方式 | 优点 | 缺点 |
|----------|------|------|
| 顺序存储 | 存储密度高，随机访问快 | 仅适合完全二叉树，否则浪费空间 |
| 链式存储（二叉链表）| 适合任意二叉树，插入删除只需改指针 | 需额外指针域，无法随机访问 |

- n个结点的二叉链表有 **n+1** 个空指针域

### 4.7 哈夫曼树（Huffman Tree）★
- **用途**：数据压缩（最优前缀编码）
- **特点**：只有度为0和度为2的结点，没有度为1的结点
- **公式**：$n_0 = n$（叶子数），则总结点数 = $2n - 1$
- **WPL**（带权路径长度）= 所有叶子结点的权值 × 路径长度 之和
- 构造方法：每次选两个最小权值结点合并

### 4.8 前缀码
- 任何编码都不是其他编码的前缀
- 哈夫曼编码是前缀码
- 例：`(0,1,00,11)` 不是前缀码（0是00的前缀）

### 4.9 树与二叉树的转换
- 任何普通树可唯一转换为二叉树（左孩子右兄弟法）
- 树 ≠ 二叉树（二叉树有左右之分，树无此区分）
- 二叉树中即使只有一棵子树也必须区分左右

## 五、图

### 5.1 基本概念
- **路径长度**：路径上的边数；路径上的顶点数 = 边数 + 1 = k+1
- **连通图**：n个顶点的连通图至少需要 **n-1** 条边（生成树）
- **生成树**：包含所有顶点的极小连通子图，边数 = 顶点数 - 1

### 5.2 有向图度数公式
- 所有顶点的出度之和 = 所有顶点的入度之和 = 边数
- 所有顶点的**度数之和** = 出度之和 + 入度之和 = **2 × 边数**

### 5.3 图的存储结构

| 存储结构 | 空间复杂度 | 判断边 | 计算度 | 适用场景 |
|----------|-----------|--------|--------|----------|
| 邻接矩阵 | O(n²) | O(1) | O(n) | 稠密图 |
| 邻接表 | O(n+e) | O(d) | O(n)（入度需遍历）| 稀疏图 |

- 无向图邻接矩阵：**对称**
- 有向图第i行1的个数 = 顶点i的**出度**
- 无向图邻接表：边结点数是边数的**2倍**

### 5.4 图的遍历
- **DFS**（深度优先）：类似树的先序遍历，借助**栈**（递归）
- **BFS**（广度优先）：类似树的层次遍历，借助**队列**
- 遍历序列**不一定唯一**（与邻接表中存储顺序有关）

### 5.5 最小生成树
- **Kruskal算法**：每次选权值最小且不构成环的边
- 最小生成树 ≠ BFS生成树（BFS不保证权值和最小）

### 5.6 拓扑排序
- 有向无环图（DAG）才有拓扑序列
- 有环图不存在拓扑序列

### 5.7 有向树
- 仅一个顶点入度为0（根），其余顶点入度均为1
- 从根到每个顶点有唯一路径

## 六、查找

### 6.1 顺序查找
- **时间复杂度**：O(n)（与表是否有序无关）
- **ASL（平均查找长度）**：$(n+1)/2$
- **哨兵**：设置在r[0]，减少越界判断，提高效率
- **适用**：小表、无序表、链表

### 6.2 折半查找（二分查找）
- **前提条件**：顺序存储 + 有序表
- **时间复杂度**：O(log₂n)
- **适用**：有序大表、静态查找
- **不适用**：无序表、链表

> 🔑 二分查找核心代码：
```python
low, high = 0, len(arr) - 1
while low <= high:
    mid = (low + high) // 2
    if x == arr[mid]: return mid
    elif x > arr[mid]: low = mid + 1
    else: high = mid - 1
return -1
```

### 6.3 二叉排序树（BST）
- 左子树 < 根 < 右子树
- **中序遍历**得到递增有序序列
- 查找平均O(log₂n)，最坏O(n)（退化为链表）
- **删除节点三种情况**：
  1. 叶子结点：直接删除
  2. 单子树：用子结点替代
  3. 双子树：用中序前驱或后继替代

### 6.4 散列表（哈希表）

#### 散列函数：除留余数法
- $H(key) = key \bmod p$，p为≤表长的**质数**（减少冲突）

#### 冲突解决

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 线性探测法 | 依次检查下一个单元 `(H+i) mod m` | 实现简单 | 易产生**堆积/聚集** |
| 二次探测法 | 探测序列 $1^2, -1^2, 2^2, -2^2, \dots$ | 缓解聚集 | — |
| 链地址法 | 同义词链成单链表 | 无聚集，删除方便 | 需额外指针空间 |
| 再散列法 | 设计多个散列函数 | — | 需设计多个函数 |

> ⚠️ **冲突无法完全避免**（鸽巢原理）

#### 开放定址法 vs 链地址法
| 方法 | 空间 | 聚集 | 删除 | 缓存 |
|------|------|------|------|------|
| 开放定址法 | 利用率高 | 有堆积 | 困难 | 命中率高 |
| 链地址法 | 需指针空间 | 无 | 方便 | 命中率低 |

## 七、排序

### 7.1 排序算法总览

| 排序算法 | 平均时间 | 最好 | 最坏 | 空间 | 稳定性 |
|----------|----------|------|------|------|--------|
| 直接插入排序 | O(n²) | O(n) | O(n²) | O(1) | ✅ 稳定 |
| 折半插入排序 | O(n²) | O(nlog₂n) | O(n²) | O(1) | ✅ 稳定 |
| 希尔排序 | 取决于增量 | — | — | O(1) | ❌ 不稳定 |
| 冒泡排序 | O(n²) | O(n) | O(n²) | O(1) | ✅ 稳定 |
| 快速排序 | O(nlog₂n) | O(nlog₂n) | O(n²) | O(log₂n) | ❌ 不稳定 |
| 堆排序 | O(nlog₂n) | O(nlog₂n) | O(nlog₂n) | O(1) | ❌ 不稳定 |

### 7.2 直接插入排序
- **思想**：将序列分成有序和无序两部分，逐个将无序元素插入有序部分
- **监视哨**：r[0]暂存待插入元素
- **最好情况**：已有序时，每趟比较1次，总O(n)
- **稳定**

### 7.3 折半插入排序
- 用折半查找确定插入位置，**减少比较次数**（比较O(log₂n)）
- **移动次数不变**（仍需移动元素）
- **稳定**

### 7.4 希尔排序
- **思想**：分割子序列 → 各自插入排序 → 逐步缩小增量 → 增量为1
- **时间复杂度**取决于**增量序列**的选择
- **不稳定**（相同元素可能分到不同子序列）

### 7.5 堆排序
- **大根堆**：每个父结点 ≥ 子结点
- **小根堆**：每个父结点 ≤ 子结点
- 属于**选择排序**
- 时间复杂度 O(nlog₂n)，最好最坏平均一致
- **不稳定**

### 7.6 排序算法稳定性总结
| ✅ 稳定 | ❌ 不稳定 |
|---------|-----------|
| 直接插入排序 | 希尔排序 |
| 折半插入排序 | 快速排序 |
| 冒泡排序 | 堆排序 |

## 八、核心公式速查表

| 公式 | 说明 |
|------|------|
| $n_0 = n_2 + 1$ | 二叉树叶子数与度为2结点数的关系 |
| 第k层最多 $2^{k-1}$ 个结点 | 二叉树每层最大结点数 |
| n个叶子→哈夫曼树共 $2n-1$ 个结点 | 哈夫曼树结点数 |
| 顺序表插入移动 $n-i+1$ 个 | 在第i个位置插入 |
| 顺序表删除移动 $n-i$ 个 | 删除第i个元素 |
| 顺序查找ASL = $(n+1)/2$ | 等概率情况 |
| 连通图至少 $n-1$ 条边 | 生成树的边数 |
| 有向图度数之和 = $2s$ | s为出度之和 |
| 路径长k→顶点数 $k+1$ | 路径顶点数 |
| n个结点的二叉链表有 $n+1$ 个空指针域 | 链式存储 |

## 九、易错点警示

1. ❌ 二叉树的度一定为2 → ✅ 二叉树的度**可以小于2**
2. ❌ 二叉树是树的特殊情形 → ✅ 树和二叉树是**两种不同**的树形结构
3. ❌ 完全二叉树一定是满二叉树 → ✅ 满二叉树一定是完全二叉树，反之不成立
4. ❌ 先序+后序可唯一确定二叉树 → ✅ **不能**，必须配合中序
5. ❌ 层次遍历借助栈 → ✅ 层次遍历借助**队列**
6. ❌ 树的度是结点度的平均值 → ✅ 树的度是结点度的**最大值**
7. ❌ 散列冲突可完全避免 → ✅ **不能**完全避免（鸽巢原理）
8. ❌ 折半查找适用于所有线性表 → ✅ 仅适用于**有序的顺序表**
9. ❌ 线性表的逻辑顺序与存储顺序总一致 → ✅ 链表的逻辑顺序与存储顺序**不一定一致**
10. ❌ 少用一个存储空间解决循环队列假溢出 → ✅ 少用一个空间是区分**队空和队满**；假溢出通过**循环队列**解决

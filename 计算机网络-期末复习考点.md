## 一、网络分层模型（必考基石）

### 1.1 两张分层表 ⭐

| 层次 | OSI 七层 | TCP/IP 四层 | PDU（数据单元） | 代表协议 | 设备 |
|:----:|----------|:----------:|:--------------:|----------|------|
| 7 | 应用层 | | | | |
| 6 | 表示层 | **应用层** | **报文** (Message) | HTTP, SMTP, FTP, DNS | |
| 5 | 会话层 | | | | |
| 4 | 传输层 | **传输层** | **报文段** (Segment) | TCP, UDP | |
| 3 | 网络层 | **网络层** | **分组/数据包** (Packet) | IP, ARP, ICMP | **路由器** |
| 2 | 数据链路层 | **网络接口层** | **帧** (Frame) | CSMA/CD, PPP | **交换机/网桥** |
| 1 | 物理层 | | **比特流** (Bit) | — | **集线器/中继器** |

> 📝 七层记忆口诀：**"物数网传会表应"**（从下往上）

### 1.2 数据封装顺序 ⭐

```
发送方（封装，从上到下）：
应用数据 → [TCP头] → 报文段 → [IP头] → 分组 → [MAC头+尾] → 帧 → 比特流

接收方（解封，从下到上）：反向逐层拆头
```

### 1.3 网络分类

| 类型 | 范围 |
|------|------|
| **LAN**（局域网） | 一栋楼/一个校园 |
| **MAN**（城域网） | 一个城市 |
| **WAN**（广域网） | 跨国家/跨洲 |

### 1.4 交换方式

> 因特网核心采用 **分组交换**（非电路交换）。

| 交换方式 | 特点 |
|----------|------|
| 电路交换 | 先建立连接再通信，独占资源（如传统电话） |
| **分组交换** | 数据分成小包独立发送，共享链路，灵活高效 |

## 三、数据链路层（必考点）

### 3.1 以太网与 CSMA/CD ⭐

| 概念 | 解释 |
|------|------|
| **CSMA/CD** | 载波监听多路访问 / 冲突检测 |
| 工作过程 | ① 先监听 → ② 空闲则发 → ③ 边发边检测冲突 → ④ 冲突则停，随机等再发 |
| **最短帧长** | **64 字节**（保证冲突检测能正常工作） |
| 冲突域 | Hub 所有端口同属一个冲突域；**交换机每个端口独立一个冲突域** |

### 3.2 交换机 ⭐

| 特性 | 说明 |
|------|------|
| **自学习** | 自动记录**源 MAC 地址 + 入端口号**的映射表（MAC 地址表） |
| **VLAN** | 虚拟局域网——逻辑分割广播域，不同 VLAN 间需路由器通信 |
| **802.1Q** | VLAN 中继标准——在帧头插入 4 字节 VLAN 标签 |
| **STP**（生成树） | 防止交换机环路（环路会导致广播风暴、MAC 表震荡） |

### 3.3 帧定界与透明传输

> **问题**：如何区分一帧的开始和结束？数据里出现了标志位怎么办？

| 方法 | 原理 |
|------|------|
| **FLAG 字节** | 用特殊标志 `01111110` 标记帧首/尾 |
| **ESC 转义法** | 数据中出现的 FLAG 前面加 ESC 转义字符 |
| **零比特填充** ⭐ | 发送方：数据比特流中连续 5 个 `1` 后自动插入 1 个 `0`；接收方：连续 5 个 `1` 后删除 1 个 `0` |

### 3.4 差错控制 —— CRC 循环冗余校验 ⭐

> **必考一道计算！** 核心：发送方在数据后加冗余码（FCS），接收方用同一个生成多项式去除——余数为 0 则传输正确。

| 要点 | 说明 |
|------|------|
| 运算方式 | **模 2 除法**（异或 XOR：同 0 异 1，无进位无借位） |
| FCS 位数 | = 生成多项式最高次幂（即除数位数 − 1） |
| 发送数据 | = 原始数据 + FCS |
| 校验判据 | 接收方用同一除数去除 → **余数 = 0 则无误** |

> 📝 详细例题见下篇"计算二"。

### 3.5 可靠传输协议

| 协议 | 原理 | 效率 |
|------|------|:---:|
| 停等协议 | 发一个帧，等 ACK；超时重传 | 低 |
| **GBN**（回退 N 帧） | 发送窗口 > 1；出错回退到出错帧，之后全部重传 | 中 |
| **SR**（选择重传） | 只重传出错的那一帧 | 高 |

## 四、网络层（分值最高）

### 4.1 路由器

> 工作在网络层，连接**不同网段**，基于 **IP 地址**进行**路由选择与分组转发**。

### 4.2 IPv4 核心考点

| 考点 | 内容 |
|------|------|
| 地址位数 | **32 位**，点分十进制表示（如 `192.168.1.1`） |
| **TTL**（生存时间） | 每经过一个路由器 **−1**，减到 0 则丢弃分组 |
| 私有地址 | `10.x.x.x`、`172.16~31.x.x`、`192.168.x.x` |
| 回环地址 | `127.0.0.1`（本机测试用） |
| 公有/私有 | 私有地址不能直接上公网，需 NAT 转换 |

### 4.3 子网划分（必考大题 ⭐⭐⭐）

**核心公式三件套：**

| 公式 | 含义 |
|------|------|
| **子网数 = 2ⁿ** | n = 从主机位借给子网的位数 |
| **每子网主机数 = 2ᵐ − 2** | m = 剩余主机位数（−2：去掉网络地址 + 广播地址） |
| **块大小 = 2ᵐ** | 相邻子网的网络地址差值 |

**C 类地址速记：**

| 掩码 | CIDR | 借位 | 子网数 | 每子网主机数 | 块大小 |
|------|:----:|:---:|:-----:|:----------:|:-----:|
| `255.255.255.128` | /25 | 1 | 2 | 126 | 128 |
| `255.255.255.192` | /26 | 2 | 4 | 62 | 64 |
| `255.255.255.224` | /27 | 3 | 8 | 30 | 32 |
| `255.255.255.240` | /28 | 4 | 16 | 14 | 16 |
| `255.255.255.248` | /29 | 5 | 32 | 6 | 8 |

### 4.4 核心协议

| 协议 | 全称 | 作用 | 工作方式 |
|------|------|------|----------|
| **ARP** ⭐ | 地址解析协议 | IP → MAC | **广播请求**"谁是 X.X.X.X？"，目标主机**单播应答** |
| **DHCP** | 动态主机配置 | 自动分配 IP/网关/DNS | DISCOVER→OFFER→REQUEST→ACK 四步 |
| **ICMP** | 互联网控制报文 | 差错报告与网络诊断 | `ping` 命令使用 ICMP 回显请求/应答 |
| NAT | 网络地址转换 | 私有地址 ↔ 公网地址 | 路由器改写 IP 包头 |

## 五、传输层（必背区分）

### 5.1 TCP vs UDP ⭐

| 对比维度 | TCP | UDP |
|----------|-----|-----|
| 连接方式 | **面向连接**（三次握手） | **无连接**（直接发） |
| 可靠性 | **可靠**（确认 + 超时重传） | **不可靠**（发出不管） |
| 首部大小 | 20 字节 | **8 字节** |
| 通信方式 | 仅**单播** | 单播 + **多播 + 广播** |
| 流量控制 | ✅ 滑动窗口 | ❌ |
| 拥塞控制 | ✅ 慢启动/拥塞避免 | ❌ |
| 适用场景 | 文件传输、网页、邮件 | 视频直播、语音、DNS |

### 5.2 端口号（高频）

| 端口 | 协议 | 助记 |
|:---:|------|------|
| **80** | HTTP | 网页 |
| **443** | HTTPS | 加密网页 |
| **21** | FTP 控制 | 文件传输 |
| **20** | FTP 数据 | 文件传输 |
| **53** | DNS | 域名解析 |
| **25** | SMTP | 发邮件 |
| **110** | POP3 | 收邮件 |

### 5.3 TCP 标志位 ⭐

| 标志 | 含义 | 出现场景 |
|:---:|------|----------|
| **SYN** | 同步序号 | 三次握手建立连接 |
| **ACK** | 确认有效 | 几乎每个 TCP 报文（除第一个 SYN） |
| **FIN** | 结束连接 | 四次挥手释放连接 |
| RST | 复位连接 | 异常断开 |
| PSH | 立即推送 | 催促接收方立即处理 |

### 5.4 三次握手与四次挥手

```
三次握手（建立连接）：
  客户端 → SYN=1, seq=x                    → 服务器
  客户端 ← SYN=1, ACK=1, seq=y, ack=x+1    ← 服务器
  客户端 → ACK=1, seq=x+1, ack=y+1        → 服务器

四次挥手（释放连接）：
  客户端 → FIN=1, seq=u                   → 服务器
  客户端 ← ACK=1, seq=v, ack=u+1          ← 服务器
  客户端 ← FIN=1, ACK=1, seq=w, ack=u+1   ← 服务器
  客户端 → ACK=1, seq=u+1, ack=w+1        → 服务器
  客户端等待 2MSL 后关闭
```

### 5.5 确认号 ⭐

> **确认号 = 期望收到的下一个字节序号**。  
> 例：收到序号 100~199 共 100 字节，确认号 = `100 + 100 = 200`。

## 计算一：子网划分 ⭐⭐⭐（分量最重）

### 题型 A：已知 IP + 掩码，求网络地址/广播地址/可用 IP

> **例题 1**：某主机 IP = `192.168.1.135`，子网掩码 = `255.255.255.192`（/26），求该主机所在网络的**网络地址、广播地址**和**可用 IP 范围**。

**解题思路：**

```
Step 1：算块大小
掩码 255.255.255.192 → 最后一段 192
块大小 = 256 − 192 = 64（每个子网 64 个地址）

Step 2：确定 135 落在哪个子网
135 ÷ 64 = 2（整数商），子网序号从 0 开始 → 子网 2
子网 2 的起始地址 = 2 × 64 = 128

Step 3：写出答案
网络地址   = 192.168.1.128           （起始）
广播地址   = 192.168.1.128 + 64 − 1 = 192.168.1.191
可用 IP    = 192.168.1.129 ~ 192.168.1.190
可用数量   = 64 − 2 = 62 个          （去掉网络地址和广播地址）
```

| 答案项 | 值 |
|--------|-----|
| 网络地址 | `192.168.1.128` |
| 广播地址 | `192.168.1.191` |
| 可用 IP 范围 | `192.168.1.129` ~ `192.168.1.190` |
| 可用主机数 | **62** |

### 题型 B：按需求划分子网

> **例题 2**：公司获得 C 类地址 `210.30.40.0/24`，需要 **5 个子网**，每个子网至少容纳 **20 台**主机。求子网掩码及各子网网络地址。

**解题思路：**

```
Step 1：确定借多少位给子网
2ⁿ ≥ 5 → n = 3（因为 2³=8 ≥ 5）
借 3 位 → 掩码 = /24+3 = /27 → 255.255.255.224

Step 2：验证主机数
剩余主机位 m = 32−27 = 5
每子网可用主机 = 2⁵ − 2 = 30 ≥ 20 ✅ 满足

Step 3：块大小 = 2⁵ = 32，每个子网依次 +32
```

| 子网 | 网络地址 | 可用 IP 范围 | 广播地址 |
|:----:|-----------|:-----------:|-----------|
| 1 | `210.30.40.0` | `.1` ~ `.30` | `.31` |
| 2 | `210.30.40.32` | `.33` ~ `.62` | `.63` |
| 3 | `210.30.40.64` | `.65` ~ `.94` | `.95` |
| 4 | `210.30.40.96` | `.97` ~ `.126` | `.127` |
| 5 | `210.30.40.128` | `.129` ~ `.158` | `.159` |

> 掩码统一为 `255.255.255.224`（/27），实际用 5 个，剩余 3 个备用。

### 子网划分速算口诀

```
块大小 = 256 − 掩码最后一段
网络地址 = 块大小 × (IP最后一段 ÷ 块大小)  ← 取整数商
广播地址 = 网络地址 + 块大小 − 1
```

## 计算二：CRC 循环冗余校验 ⭐

### 核心概念

| 术语 | 含义 |
|------|------|
| **生成多项式 G(x)** | 收发双方约定的固定除数 |
| **FCS**（帧检验序列） | 用模 2 除法算出的冗余码，附在数据尾部 |
| **模 2 除法** | 逐位异或（XOR）：相同→0，不同→1，**不进位不借位** |

### 标准步骤

```
① G(x) → 二进制（位宽 = 最高次幂 + 1）
② 原始数据 M 后面补 (位宽 − 1) 个 0 → M'
③ M' ÷ G（模 2 除法）→ 余数 R
④ R 的位数不够 (位宽 − 1) 时补前导 0
⑤ FCS = R
⑥ 发送数据 = M + FCS（拼接）
```

### 模 2 除法运算规则

```
逐位处理：
  • 当前余数最高位 = 1 → 商 1，余数 XOR 除数
  • 当前余数最高位 = 0 → 商 0，余数 XOR 0000（即不变）
  • 余数左移，从被除数拉下一位
  • 重复直到被除数所有位处理完
  • 最终余数 = FCS（去掉多于 (除数位数−1) 的高位 0）
```

---

> **例题 3**：发送数据 M = `101001`，生成多项式 G(x) = x³ + x² + 1，求冗余码 FCS 及发送方实际发出的完整数据。

**解答：**

```
Step 1：G(x) = x³ + x² + 0·x + 1 → 二进制 1101（4 位）

Step 2：FCS 位数 = 4 − 1 = 3
       被除数 M' = 101001 + 000 = 101001000（补 3 个 0）

Step 3：模 2 除法（除数 1101）
```

模 2 除法竖式推演：

| 步骤 | 当前被除数段 | 最高位？ | XOR 操作 | 结果 | 拉下一位 | 新被除数段 |
|:----:|:----------:|:------:|----------|:----:|:------:|:----------:|
| 1 | `1010` | 1 | `1010⊕1101` | `0111` | 0 | `1110` |
| 2 | `1110` | 1 | `1110⊕1101` | `0011` | 1 | `0111` |
| 3 | `0111` | 0 | 不 XOR | `0111` | 0 | `1110` |
| 4 | `1110` | 1 | `1110⊕1101` | `0011` | 0 | `0110` |
| 5 | `0110` | 0 | 不 XOR | `0110` | 0 | `1100` |
| 6 | `1100` | 1 | `1100⊕1101` | `0001` | — | **余数 `001`** |

> 📝 步骤 6 后没有更多位了，最终余数为 `0001`（4 位）。由于 FCS 只需 3 位，取低 3 位 → FCS = **001**（或将 `0001` 去掉前导 0，即为 `001`）。

```
Step 4：FCS = 001

Step 5：发送数据 = 101001 + 001 = 101001001
```

> 📝 **验证**：接收方用 `101001001 ÷ 1101` 做模 2 除法，余数为 `000` → 传输无误。

## 计算三：TCP 序号与确认号 ⭐

### 核心规则

| 概念 | 公式/规则 |
|------|-----------|
| **序号（SEQ）** | 本报文段**第一个数据字节**的编号 |
| **确认号（ACK）** | 期望收到的**下一个字节**序号 = 已收到最后一个字节序号 + 1 |
| 三次握手 SYN 消耗 | SYN 报文**消耗一个序号**（虽然不传数据） |
| 确认号示例 | 收到 seq=100、len=50 的数据 → ack = 150 |

---

> **例题 4**：主机 A 向主机 B 发送数据。已知三次握手中 A 的初始序号为 **200**，B 的初始序号为 **500**。建立连接后，A 向 B 发送一个 **100 字节**的数据段。请写出整个过程中每一步的 SEQ 和 ACK。

**解答：**

```
① 三次握手：

Step 1: A→B   SYN=1, seq=200
               （A 发起，消耗一个序号）

Step 2: B→A   SYN=1, ACK=1, seq=500, ack=201
               （B 确认收到的200，期望下一个是201）

Step 3: A→B   ACK=1, seq=201, ack=501
               （A 确认 B 的500，期望501；连接建立）

② 数据传输：

Step 4: A→B   seq=201, data=100字节
               （A 发送100字节数据，占用序号201~300）

Step 5: B→A   ACK=1, ack=301
               （B 确认收到，期望下一个字节是301）
```

| 步骤 | 方向 | 内容 | SEQ | ACK |
|:---:|:---:|------|:---:|:---:|
| 握手1 | A→B | SYN | 200 | — |
| 握手2 | B→A | SYN+ACK | 500 | 201 |
| 握手3 | A→B | ACK | 201 | 501 |
| 数据 | A→B | 100字节 | 201 | — |
| 确认 | B→A | ACK | — | **301** |

## 计算四：以太网最短帧 & 冲突检测

### 核心概念

| 概念 | 含义 |
|------|------|
| **最短帧长** | 64 字节（以太网规定，含头部和 CRC） |
| 64 字节的来源 | 保证发送方在**发完最短帧之前能检测到冲突** |
| **时隙** | 以太网冲突窗口 = 512 位时（64 字节 × 8 bit/byte） |
| 关键不等式 | **发送时间 ≥ 2 × 传播时延**（即信号走一个来回的时间） |

### 计算关系

```
最短帧长（位）= 带宽 × 2 × 传播时延
最短帧长（字节）= 最短帧长（位）÷ 8
```

---

> **例题 5**：某以太网带宽为 100 Mbps，两站点最远距离为 2 km，信号传播速度为 2×10⁸ m/s。问该以太网的最短帧长应为多少字节？是否满足以太网标准的 64 字节要求？

**解答：**

```
Step 1：算单程传播时延
传播时延 = 距离 ÷ 速度 = 2000 m ÷ (2×10⁸ m/s) = 10⁻⁵ s = 10 μs

Step 2：算冲突窗口（来回时延）
2 × 传播时延 = 2 × 10 μs = 20 μs

Step 3：算最短帧长（位）
最短帧长 = 带宽 × 2 × 传播时延 = 100×10⁶ bps × 20×10⁻⁶ s = 2000 bit

Step 4：转字节
最短帧长 = 2000 ÷ 8 = 250 字节

Step 5：比较
250 字节 > 标准 64 字节 ✅ 标准 64 字节在此网络下不够用
→ 需要增大最短帧长或缩短距离
```

> 📝 **公式速记**：最短帧长（字节）= (带宽 × 2 × 距离 / 信号速度) / 8

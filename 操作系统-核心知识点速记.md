## 二、进程管理核心概念

### 2.1 进程三态转换 ⭐

```
         ┌─────────┐
    ┌───→│  就绪态  │←──────┐
    │    └────┬─────┘       │
    │  调度   │   事件完成    │
    │    ┌────↓─────┐       │
    │    │  运行态   │───事件等待──→┌────────┐
    │    └────┬─────┘              │ 阻塞态  │
    │    时间片到                  └────────┘
    └─────────┘
```

> 🚫 **绝对不能**：阻塞 → 运行（必须先回就绪排队！）

### 2.2 fork() 一句话

> **一次调用，两次返回**：父进程得子 PID（>0），子进程得 0。  
> 不调 `wait()` 则输出顺序**随机**。

## 三、调度算法四兄弟对比

| 算法 | 抢占？ | 选谁 | 优点 | 缺点 |
|------|:------:|------|------|------|
| **FCFS** | ❌ 非抢占 | 先来的 | 简单公平 | 短作业吃亏 |
| **SJF** | ❌ 非抢占 | 最短的 | 平均等待最短 | 长作业可能饿死 |
| **RR** | ✅ 抢占 | 轮流每人一个 q | 公平响应快 | q 太小切换多 |
| **优先级** | 可抢可不抢 | 优先级高的 | 紧急任务优先 | 低优先级可能饿死 |

> **周转时间公式（必背）**：T周转 = T完成 − T到达  
> **平均周转时间** = 所有人周转时间之和 ÷ 进程数

## 四、银行家算法 —— 四步走

```
① 算 Need = Max - Allocation（逐位对齐相减）
② 初始化 Work = Available
③ 找一个 Need ≤ Work 的进程（找不到→不安全）
④ 假设它完工，Work += 其 Allocation，标记完成，回到③
   直到全部标记 → 安全！记录序列。
```

> 💡 **类比记忆**：银行家 = OS，客户 = 进程，钱 = 资源。  
> 不放贷给还不起的人，保证所有人最终都能还钱。

## 五、信号量与 P/V 操作

### 5.1 核心规则

| | P(S) | V(S) |
|------|------|------|
| 操作 | S = S − 1 | S = S + 1 |
| 若结果 | S < 0 → **阻塞自己** | S ≤ 0 → **唤醒一个等待者** |
| 物理意义 | 申请/消耗资源 | 释放/归还资源 |

### 5.2 S 的值解读

| S 的值 | 含义 |
|:------:|------|
| S > 0 | 有 **S 个**资源可用 |
| S = 0 | 资源刚好用完，无人排队 |
| S < 0 | 资源透支，有 **\|S\| 个**进程在排队阻塞 |

### 5.3 生产者-消费者 —— 三种信号量速记

| 信号量 | 初值 | 谁 P 它 | 谁 V 它 |
|:------:|:----:|:------:|:------:|
| `empty` | N | 生产者（申请空位放货） | 消费者（取走货腾出空位） |
| `full` | 0 | 消费者（申请取货） | 生产者（放入货通知有货） |
| `mutex` | 1 | 两者都要（锁缓冲区） | 两者都要（解锁） |

> ⚠️ **死锁陷阱**：必须先 P(empty/full) 再 P(mutex)！  
> 先锁门再申请空位 = 缓冲区满时自己锁死自己。

### 5.4 实际场景映射

| 经典模型 | 电商秒杀 |
|----------|----------|
| 生产者 | 前端 Web 服务器（接收下单） |
| 缓冲区 | 消息队列 MQ（RabbitMQ/Kafka） |
| 消费者 | 后台数据库写入系统 |
| empty 初值 N | 队列最大容量（如 1000） |
| full 初值 0 | 初始队列为空 |

## 六、存储器管理

### 6.1 动态分区分配三算法

| 算法 | 链表排序 | 挑哪个 | 副作用 |
|------|:------:|--------|--------|
| **首次适应 FF** | 地址↗ | 第一个够大的 | — |
| **最佳适应 BF** | 容量↗ | 最小够用的 | 产生大量微小碎片 |
| **最坏适应 WF** | 容量↘ | 最大那块 | 剩的仍很大，碎片少 |

### 6.2 分页地址变换 —— 三步法 ⭐

```
逻辑地址 A ──┬──→ 页号 P = A / L
             └──→ 偏移量 W = A % L
查页表：P → 物理块号 b
物理地址 = b × L + W
```

> 📝 **单位陷阱**：1KB = 1024 字节，不是 1000！

## 七、页面置换算法（虚拟内存）

### 7.1 三种算法一句话

| 算法 | 看什么 | 淘汰谁 | 能否实现 |
|------|--------|--------|:------:|
| **OPT** | 未来 | 最晚才用的 | ❌ 不能 |
| **FIFO** | 入驻时间 | 最早来的 | ✅ 能 |
| **LRU** ⭐ | 最近使用 | 最久没碰的 | ✅ 能 |

### 7.2 LRU 推演心法

```
访问每一页时：
  命中？ → 把该页提到"最新"位置
  缺页？ → 踢最底下的"最老"页，新页放最顶"最新"位置
```

## 通关一：进程状态转换

**题目特征**：给一个场景，让你判断进程状态怎么变的。

**解题思路**：
1. 先看进程当前是什么状态
2. 再看发生了什么事件
3. 按转换规则判断目标状态

**常见考题：**
- "进程在等待 I/O 完成时处于什么状态？" → **阻塞态**
- "I/O 完成后进程进入什么状态？" → **就绪态**（不是运行态！）
- "时间片用完，进程从运行态变为？" → **就绪态**
- "进程被 V 操作唤醒后进入？" → **就绪态**

---

## 通关二：调度算法计算

**题目特征**：给一张进程表（到达时间 + 服务时间），让你用指定算法算周转时间、平均周转时间。

**通用解题步骤：**

```
① 画时间轴，标出每个进程的到达时刻
② 按算法规则，一步步决定每个时刻谁上 CPU
③ 记录每个进程的开始时刻和完成时刻
④ 算周转时间 = 完成 - 到达（注意：用到达时刻不是 0）
⑤ 求平均值
```

**三个算法判断要诀：**

| 算法 | 什么时候换人 |
|------|------------|
| **FCFS** | 当前进程**跑完**才换（非抢占） |
| **SJF** | 当前进程**跑完**，在已到达的里面**挑最短的** |
| **RR** | **时间片用完**就换，新来的**排在前面** |

## 通关三：银行家算法

**题目特征**：给 Available、Max、Allocation 三个矩阵，判断系统是否安全，找安全序列。

**标准解题模板：**

```
Step 1: 算 Need[i] = Max[i] - Allocation[i]（逐位相减）
Step 2: Work = Available，全部进程标记 Finish = false
Step 3: 循环查找：
        找一个 Finish=false 且 Need[i] ≤ Work 的进程 i
        找到 → 假设它完工：Work += Allocation[i]，Finish[i]=true，记录 i
        找不到 → 跳出
Step 4: 全部 Finish=true → 安全，序列就是记录的顺序
        有人 Finish=false → 不安全，死锁风险
```

**关键技巧**：
- Need ≤ Work 的判断是**逐位比较**，三个资源都要满足
- 每轮可能多条路，任选一条走得通就行
- 多轮循环直到全部标记或卡住

## 通关四：P/V 操作与生产者-消费者

**题目特征**：给一段代码让你填空（填 P/V），或分析会不会死锁。

**解题口诀：**

```
生产者：P(empty) → P(mutex) → 干活 → V(mutex) → V(full)
消费者：P(full)   → P(mutex) → 干活 → V(mutex) → V(empty)

记忆：先"同步"（占坑），后"互斥"（锁门）
      先 P 后 V，先来的信号量先 P
```

**判断死锁题方法**：
1. 看 P(mutex) 和 P(empty/full) 的顺序
2. 如果 P(mutex) 在 P(empty) 之前 → **死锁！**
3. 原因：缓冲区满时，生产者先锁门再等空位，消费者被挡在门外无法取货腾空位

## 通关五：动态分区分配

**题目特征**：给空闲分区表，某进程申请 n KB，问各算法分配哪个分区。

**三种算法查找顺序：**

```
FF（首次适应）：从低地址往高地址挨个看，选第一个 ≥ n 的
BF（最佳适应）：所有分区按大小排好序，从小到大找第一个 ≥ n 的
WF（最坏适应）：所有分区按大小排好序，从大到小找第一个 ≥ n 的
```

> 💡 **记法**：FF = 按地点找（First Fit），BF = 找最紧的（Best Fit），WF = 找最松的（Worst Fit）

## 通关六：分页地址变换

**题目特征**：给页表 + 页面大小 + 逻辑地址，求物理地址。

**死套公式三步：**

```
已知：逻辑地址 A，页面大小 L
① P = A / L（整除，取整数部分）
② W = A % L（取余数）
③ 查页表：页号 P → 块号 b
④ 物理地址 = b × L + W
```

> ⚠️ **最容易错**：单位不统一！1KB = 1024 字节。  
> 如果题目说的是"页面大小 2KB"，L = 2048。  
> 如果题目说的是"页面大小 4KB"，L = 4096。

## 通关七：页面置换算法（LRU）

**题目特征**：给页面引用序列 + 物理块数，画格子图推演缺页过程。

**LRU 推演标准步骤：**

```
① 准备一张表：行=物理块(从上到下:新→老)，列=每个页面引用
② 逐列处理每个页面引用：
   - 如果该页已在内存中（命中）：
       把该页移到最上面（最新位置），其他页相对顺序不变
       标记 ✔
   - 如果该页不在内存中（缺页）：
       内存不满 → 直接放最上面
       内存已满 → 踢掉最下面的（最老），新页放最上面
       标记 ❌
③ 数有几个 ❌ 就是缺页次数
```

**手动推演要点**：
- 每次操作后，内存从**上到下**一定严格是 **最新→最老**
- 命中的页一定要"翻新"（移到最顶部）
- 缺页时踢的一定是**最底下**那个（最老的）
